# 888 · Living Reverie

> **The one question:** What if Karel's own recorded piano became a 10-minute
> _living_ reverie — genuinely different at minute 8 than at minute 1 — a
> generative engine with **memory** that re-voices and re-orchestrates his
> motifs across an irreversible arc (sparse → blooming → dense → dissolving),
> never looping?

This is the adult, cinematic, long-form register of the lab: not a toy you poke,
but a piece you let unfold. Press **Begin the reverie** and walk away; ten
minutes later it has arrived somewhere it can never return from.

---

## The two-tier memory engine (`engine.ts`)

The design is anchored in a 2026 research finding: a **single** memory mechanism
structurally cannot hold both timescales at once. Local melodic continuity
("this minute sounds like the last") and a global arc ("the whole piece goes
somewhere and doesn't come back") are different problems and need **two explicit
tiers**. So the engine implements exactly that.

### Local tier — the pitch-cell bank

A small bank (≤14) of recent **pitch cells**. Each cell is a short motif stored
as *scale degrees + rhythm + velocity* — deliberately abstract, not absolute
pitches, so it can be re-quantized into any mode/tonic later.

- New phrases are generated by **picking a recent cell and lightly varying it**:
  transpose by a scale degree or two, octave-shift, nudge the tempo and
  dynamics. Because every phrase descends from a neighbour already in the bank,
  minute _N_ audibly resembles minute _N − 1_ — that is the local continuity.
- Variations are occasionally folded **back** into the bank, so the material
  slowly drifts rather than staying frozen — memory with gentle mutation.
- **Live seeding:** when Karel's recording loads, the engine **chroma-folds**
  the live audio — it reads the spectrum, accumulates magnitude into 12
  pitch-class bins, picks the loudest classes, and maps them to the nearest
  scale degrees of the current mode to mint a fresh cell every few seconds. His
  actual playing keeps re-seeding the melodic DNA.
- **Fallback seeding:** with no network, the bank starts from a built-in
  "Welcome Home"-flavoured motif and evolves entirely on its own.

### Global tier — irreversible age + state machine

One monotonic `age` value walks `0 → 1` across `AGE_DURATION_SEC = 600` (ten
minutes). Age is **irreversible** — it only ever increases and never loops. It
drives a four-section state machine:

| Section | Age range | Character |
|---|---|---|
| `sparse` | 0.0 – 0.2 | long silences, soft single voices, low register |
| `blooming` | 0.2 – 0.5 | phrases tighten, pad swells, shimmer enters |
| `dense` | 0.5 – 0.8 | fastest note rate, widest register, fullest texture |
| `dissolving` | 0.8 – 1.0 | density collapses, dynamics fade toward stillness |

Age (and the section it implies) controls **voice density, note rate, register
spread, reverb depth, and brightness** — so the piece is demonstrably a
different world late versus early.

### Modal journey

As age advances the engine walks the mode
**Ionian → Lydian → Mixolydian → Dorian → Aeolian**, with a slowly creeping
tonic (a few semitones up across the whole piece). Every synthesized note is
pitch-quantized to the **current** mode over the **current** tonic, so the
harmonic light shifts from bright/open early to darker/inward late without any
seam.

## Synthesis & master chain (`audio.ts`)

- **FM-ish piano voice** — sine carrier + sine modulator at an inharmonic ratio,
  fast-decay amplitude envelope; modulation depth (brightness) scales with age.
- **Detuned-saw pad** — two saws ±7 cents through a lowpass, slow swell, for
  sustained modal harmony.
- **Granular shimmer** — short panned triangle grains an octave up, sprinkled
  more densely as the piece blooms.
- **Master chain:** `voices → ConvolverNode reverb → DynamicsCompressor →
  destination`. The reverb impulse response is **built at runtime** from an
  exponentially-decaying noise burst rendered in an `OfflineAudioContext` — no
  external IR file. Reverb **wet grows with age**, so the space deepens and
  widens as the reverie ages.

It is **sounding and moving within ~1.5 s** using the fallback — nothing waits on
the network to begin.

## His real piano + the fallback

Recording ID `549fc519-f7fc-4c38-a771-adaad2edbc81`, fetched from the read-only
`/api/audio/:id` route (no API route is created here; all code is client-side).

- If the buffer loads, it is played **quietly under** the generative voices and
  tapped by an analyser. After ~350 ms, if there is non-zero energy the status
  flips to emerald **"playing Karel's piano"** and the live chroma starts
  re-seeding the local memory bank.
- On **any** failure (offline, decode error, timeout) the built-in generative
  engine runs with **zero network**, and the status shows amber **"generative
  piano — live recording unavailable."** Either way it is fully sounding and
  moving.

## Visual mapping — a morphing displaced membrane (`scene.ts`)

Deliberately **not** a particle field (the lab already has one). The visual is a
single high-resolution subdivided icosphere whose vertices are displaced in a
**vertex shader** by layered 3D simplex noise (fbm). **The mesh is the arc:**

| Section | Membrane |
|---|---|
| `sparse` | nearly flat, dark, slow undulation |
| `blooming` | ridges rise, color warms, gentle outward growth |
| `dense` | folded, turbulent terrain; fullest palette; fastest motion |
| `dissolving` | amplitude collapses back to flat; cools to deep violet; fades |

Displacement amplitude, noise frequency, palette, emissive glow and motion speed
are all driven by `age`, the current section, and a smoothed per-frame audio RMS
(so it breathes with the sound). An additive back-side glow shell plus
exponential fog give a bloom-like look with **no postprocessing dependency**. The
palette re-voices across the arc: cool indigo → warm amber/rose at bloom → deep
violet at dissolve. **Drag to orbit** the camera; it also auto-drifts.

WebGL cleanup is mandatory and complete: every geometry/material is tracked in a
`disposables` array; on unmount we dispose all resources, call
`renderer.dispose()` + `renderer.forceContextLoss()`, remove listeners, stop all
audio voices, and `audioCtx.close()`. WebGL-unavailable shows a readable notice
instead of crashing.

## Multi-cycle commitment — next-cycle deepening

This is the lab's long-form skeleton (cycle 1 of a multi-cycle thread). It shipped
as the winner of a DEEP fire (cycle 531) that built the SAME two-tier engine
against three different visual languages. The two banked siblings are the deepening
backlog — fold them in on later cycles:

- **Volumetric raymarch twin (`889`, banked):** the arc rendered as a raymarched
  FBM nebula on a fullscreen quad — wisps → bloom → luminous turbulence → cooling
  dissolve. More cinematic on a desktop GPU; de-selected here on mobile-GPU risk
  for the unattended 06:30 phone glance. Resurrect as an optional "volumetric"
  render mode toggled at runtime.
- **Canvas2D ink-strata twin (`890`, banked):** the arc as accumulating
  ink-in-water chroma strata with a feedback smear — zero GPU, the most bulletproof
  fallback render. Resurrect as the WebGL-unavailable visual path so the piece
  always has a living visual.
- **Engine deepening (all twins agreed on this gap):** the local tier currently
  re-derives phrases from the bank rather than carrying *exact* phrase state across
  the section boundary — a true seamless hand-over (carry the in-flight motif, not
  just the bank) is the next musical step. Phrase scheduling is also wall-clock
  (`setTimeout`) rather than sample-accurate; move to look-ahead AudioContext
  scheduling for tighter cells.
