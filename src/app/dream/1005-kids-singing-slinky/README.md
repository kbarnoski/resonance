**For**: kids (4+)

Singing Slinky lets a small child flick a glowing rainbow spring in true 3D and watch a bright compression pulse race down the coil, bounce off the far end, and settle into a humming standing wave they both see and hear. The toy is built on a *longitudinal* (compression) wave, not a transverse string: each coil is a point-mass whose displacement runs **along** the helix axis, so flicking it makes coils visibly bunch up and stretch apart. No reading, no wrong notes, no scary sounds — and it flicks itself after about two seconds of stillness so it is never static or silent.

**The physics.** The slinky is a 1-D mass-spring chain integrated on the CPU with the discrete wave equation `a_i = (k/m)·(u_{i+1} − 2·u_i + u_{i−1}) − damping·v_i`, fixed timestep with substeps for stability, and both ends pinned (fixed–fixed). Fixed–fixed boundaries make the standing-wave modes a clean integer harmonic series `f_n = n·c/2L`, which is inherently consonant. A "flick" injects a Gaussian compression pulse that propagates, reflects off both boundaries, and after a few bounces relaxes into stationary nodes/antinodes. The renderer maps each coil onto a 3-D helix and nudges its axial position by `u_i`, so compression is literally visible as bunched/stretched bands.

**The audio.** The displacement field is projected onto the fixed–fixed mode shapes `sin(nπi/N)` (n=1..8); those mode amplitudes drive an additive oscillator bank tuned to the harmonic series, so the timbre swells when excited and decays as the wave damps — the fundamental literally is the spring's resonance. The grab point snaps the fundamental to D major pentatonic so any flick is musical, an always-on breathing drone keeps it from ever going silent, and the mandatory kids-safe master chain (`gain ≤0.26 → lowpass ~6500 Hz → DynamicsCompressor −10/20:1 → destination`) with soft attacks and per-voice normalization guarantees nothing harsh, shrill, or sudden.

**References.** Stefan Bilbao, *Numerical Sound Synthesis* (FDTD / finite-difference physical modeling of strings, bars, and springs); the classic Physics Classroom slinky demonstration of longitudinal-vs-transverse waves; standard treatments of longitudinal standing-wave modes of a helical spring and the harmonic series of a fixed–fixed medium.

**What's unverified.** This was authored in a headless build box with no audio device and no GPU, so the actual sound output and the live WebGL2 render were not heard or seen here — the kids-safe gain staging, the modal-amplitude-to-loudness mapping, the helix/camera framing, and the physics tuning constants (k, damping, flick strength) are designed to be in a good range but may want a small in-device pass. The Canvas2D fallback path and the iOS AudioContext gesture-gate were written to spec but likewise not exercised on real hardware here.
